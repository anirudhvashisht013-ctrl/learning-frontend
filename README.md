# Learning Lab

A personal learning notebook. Lessons and quizzes from concepts I pick up while building real products — kept in one place, indexed two ways, easy to drill on.

## What this app is

A Next.js app that runs locally and stores everything in JSON files (plus a sprinkle of `localStorage` for your view preferences and quiz progress). No backend, no auth, no accounts. Just a notebook that looks like notebook paper.

Every lesson has the same shape: a plain-language concept, a real code example, alternative approaches, and a one-paragraph critique. Every lesson is also tied to a **topic** (Tailwind, React, …) and a **codebase** (the project you encountered it in), which lets you browse the same data two ways.

## How to run it

```bash
npm install
npm run dev
```

Then open [http://localhost:3000](http://localhost:3000).

To build for production:

```bash
npm run build
npm start
```

## The two views

There's a toggle in the top bar that swaps between:

1. **By topic** (default) — tabs are topics. See every lesson that belongs to Tailwind, React, etc. URL: `/topics/tailwind`.
2. **By codebase** — tabs are projects. See every lesson that came from `slotfusion` or `learning-lab`, regardless of topic. URL: `/codebases/slotfusion`.

The two views read the same `data/lessons.json` — each lesson is double-indexed by `topicSlug` and `codebaseSlug`. The toggle persists across reloads.

## The Work-Driven Learning Loop

The whole point of this notebook: **don't learn frontend in the abstract. Learn it from the actual code you're shipping.**

The loop:

1. While building a real feature, hit something you don't fully understand (a Tailwind utility, a React pattern, a docker command).
2. Stop. Ask your AI partner to write a Learning Lab lesson about that specific thing, *in the context of where you saw it*.
3. The AI returns a JSON entry shaped like the others — concept, example, alternatives, critique, plus 3-5 quizzes.
4. Paste it into `data/lessons.json` and `data/quizzes.json`. Run `npm run dev`.
5. A few days later, hit the Quiz tab and drill yourself on what you saved.

The notebook is never edited by hand — that's the rule. AI-generated, human-pasted. Keeps content quality even and removes the temptation to write half-finished entries.

## Adding a new lesson — sample prompt

Paste this into your AI assistant (Claude, ChatGPT, etc.), filling in the bracketed bits:

> I just hit a concept while building [codebase name]. The concept is [what you saw]. I want a Learning Lab entry for it.
>
> Write a JSON object matching the `Lesson` type in `types/index.ts`:
> - `topicSlug`: one of [tailwind, react, nextjs, shadcn, typescript, backend-basics, docker, git, apis, design-patterns]
> - `codebaseSlug`: [codebase name]
> - `title`: short, scannable
> - `concept`: 3-5 sentences, plain language, no jargon
> - `example`: real code, with optional annotation
> - `alternatives`: 2-3 other approaches and when each applies
> - `critique`: one paragraph on the trade-off
> - `tags`: 2-4 words
>
> Then write 3-5 quizzes for it (mix of recall / application / critique) matching the `Quiz` type.
>
> Match the tone of the existing lessons in `data/lessons.json` — warm, direct, written for a senior product designer who is learning to code.

Paste the lesson into `data/lessons.json` and the quizzes into `data/quizzes.json`. Done.

## File structure

```
learning-lab/
├── app/                   ← Next.js App Router pages
├── components/            ← top-bar, lesson-view, quiz-runner, etc.
│   └── ui/                ← button, badge, input, dialog primitives
├── lib/                   ← data loaders, search, storage, filter
├── data/                  ← topics, codebases, lessons, quizzes (JSON)
└── types/                 ← shared TypeScript types
```

Every file aims for under ~120 lines, one concern per file.

## Keyboard shortcuts

- `⌘K` / `Ctrl+K` — open search
- `↑` / `↓` in search — move between results
- `Enter` in search — jump to the result
- `Esc` — close search

## A note on content

Lessons are added by pasting prompts generated by my AI partner — never edited by hand. This keeps content quality high and consistent.
